Input Files Formatting Guide

This page describes the format of input files for the SASI Gridder program. A set of example inputs is included with this documentation: example inputs .

General Notes

  • Column names are case-sensitive.

Overall Input Layout

The SASI Gridder takes a single directory as an input. This directory should have the following layout: 
        /
         - gear_mappings.csv
         - raw_efforts.csv
         - grid /
           - grid.dbf
           - grid.prj
           - grid.shp
           - grid.shx
         - stat_areas /
           - stat_areas.dbf
           - stat_areas.prj
           - stat_areas.shp
           - stat_areas.shx
The format of each section of this directory will be described below.

gear_mappings.csv

Example: gear_mappings.csv
This file contains mappings from trip_type codes to SASI gear codes. For example, 'otter == GC10'. This file should be a CSV file with the following columns: 
trip_type, gear_code

raw_efforts.csv

Example: raw_efforts.csv
This file contains raw efforts, typically generated as the 'vtr_a' output of Chad's 
SAS/Omnibus/VA/wet_model/a_create_vtr_a_macro.txt
script. This file should be a CSV file with the following columns: 
nemarea, trip_type, A, hours_fished, value, year, lat, lon

grid

Example: grid
This directory contains the shapefile set for the SASI 100km cell grid. The shapefile should have an 'ID' attribute that contains a numerical cell id. Other attributes will be ignored.

stat_areas

Example: stat_areas
This directory contains the shapefile set for statistical areas to be used for gridding. The shapefile should have a 'SAREA' attribute that contains a numerical stat area ID. These SAREAs should correspond to to the 'nemarea' column in the raw_efforts.csv file. Other attributes will be ignored.
